# Review Remediation

## Scope

Fix the concrete bugs, dead code, and hotspots found in the 2026-07 codebase review. Nine slices live in the sibling JSON. `blockedBy` may reference task ids from `test-system-overhaul.json`; the DAG is global across both task files.

## Decisions

- Upscale is removed whole (front-end implementation, both schemas, router stub, UI branches, dead config knobs). Confirmed by the project owner over restoring it; consistent with the aggressive-retire rule. If upscale returns, it will be rebuilt on the model-registry architecture.
- Double sRGB decode fix: prefer deleting the two `srgb_to_linear` calls in `src/lib/gpu/wgsl/develop/geometry.wgsl` (its input is always the linear output of the inputDecode pass) after re-checking every geometry call site; the alternative is restoring the inputDecode/geometry mutual exclusion in `src/lib/gpu/orchestrator.ts::buildDevelopPasses`. Presets are re-tuned on the corrected baseline after the fix (owner-confirmed); no compensation curve on top of the bug.
- Signed chat image URLs: snapshots store asset references, not token-bearing URLs; the conversation projection signs fresh URLs via the asset service. Pre-launch, so no dual-read compatibility path.
- Cleanup boundaries: `dead-code-and-hygiene` deletes only code with no future consumer; the persistence-gateway reset and the pseudo-batch `upsertElementsInWorkbench` belong to `workbench-data-correctness` (they get wired or truly merged, not deleted); upscale has its own slice.
- Key fixes (`decode-double-srgb-fix`, `persistence-correctness`, `workbench-data-correctness`) follow the new test-authoring rules once `test-authoring-rules` lands: behavior tests by a non-implementer agent, adversarial cases first.

## Knip backlog

Removing the `api/**` and broad `src/lib/gpu/**` knip entry exemptions in `dead-code-and-hygiene` exposed pre-existing dead code that the config had been hiding. The increase is exposure, not newly introduced dead code: every file deleted in that slice had zero referrers (grep-verified) before removal, so no deletion orphaned a consumer. Triage:

- `upscale-removal` owns: `shared/imageUpscaleSchema.ts` re-exports, `server/src/shared/imageUpscaleSchema.ts`, and related dead exports.
- Keep list (recorded rationale): shadcn spare components under `src/components/ui/` — the project standardizes on shadcn for new basic components, so unused library parts stay.
- `knip-backlog-cleanup` owns the rest: dead server files (`src/gateway/router/fallback.ts`, `src/providers/referenceImages.ts`, `src/shared/dataUrl.ts` under `server/`), the retired canvas panels (`CanvasPropertiesPanel`, `CanvasStoryPanel` and their model hooks), `src/data/presets.ts`, `src/lib/ai/errors.ts`, gpu orphans (`passes/utility/bilateralScale.ts`, `passes/utility/downsample.ts`, `webgpu.d.ts` vs `@webgpu/types` — one of the two type sources is redundant), `src/lib/layerMaskTexture.ts`, dead exports (mostly pass `*Options`/`*Params`/`*Handle` types exported but only used internally), and dependency triage (`@ai-sdk/*`, `ai`, `@tanstack/react-query`, `react-markdown`, `remark-gfm` reported unused at the root workspace — verify whether they are genuinely unreferenced or misattributed across the server workspace boundary before removing).
- Related trap found while building the render-chain goldens: v1 film profiles (`src/data/filmProfiles.ts`, e.g. `film-neutral-v1`) resolve through `resolveRenderProfileFromState` to a fully neutral v3 profile — LUT null, grain disabled — with no error. Only `stock-*` v2 profiles produce real film output today. No live code references v1 ids (`src/data/presets.ts` is already dead), but `getBuiltInFilmProfile` still registers them, so any future v1 id usage silently renders neutral. Decide the fate of the v1 profile data (repair the v1→v3 module mapping or retire the ids) together with the `presets.ts` cleanup.
